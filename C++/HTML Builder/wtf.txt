							<p>
								A
									<b>website</b>

									<b>website</b>

								) is a collection of
									<b>website</b>

									<b>website</b>

									<a href="/wiki/Web_page" title="Web page">web pages</a>
									<a href="/wiki/Domain_name" title="Domain name">domain name</a>
								 and published on at least one
									<b>website</b>
									<b>web site</b>
									<a href="/wiki/Web_page" title="Web page">web pages</a>
									<a href="/wiki/Domain_name" title="Domain name">domain name</a>
									<a href="/wiki/Web_server" title="Web server">web server</a>
								. Notable examples are
									<b>website</b>

								All publicly accessible websites collectively constitute the

							<p>
								The
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
								 (WWW) was created in 1990 by the British CERN physicist
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
								.
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
								 On 30 April 1993,
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
									<a href="/wiki/CERN" title="CERN">CERN</a>
								 announced that the World Wide Web would be free to use for anyone.
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
									<a href="/wiki/CERN" title="CERN">CERN</a>
									<sup id="cite_ref-w3c_2-0" class="reference">
										<a href="#cite_note-w3c-2">&#91;2&#93;</a>
									</sup>
								 Before the introduction of the
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
									<a href="/wiki/CERN" title="CERN">CERN</a>
									<sup id="cite_ref-w3c_2-0" class="reference">
										<a href="#cite_note-w3c-2">&#91;2&#93;</a>
									</sup>
									<a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a>
								 (HTTP), other protocols such as
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
									<a href="/wiki/CERN" title="CERN">CERN</a>
									<sup id="cite_ref-w3c_2-0" class="reference">
										<a href="#cite_note-w3c-2">&#91;2&#93;</a>
									</sup>
									<a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a>
									<a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">File Transfer Protocol</a>
								 and the
									<a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>
									<a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>
									<sup id="cite_ref-First_page_1-0" class="reference">
										<a href="#cite_note-First_page-1">&#91;1&#93;</a>
									</sup>
									<a href="/wiki/CERN" title="CERN">CERN</a>

									</sup>

								 were used to retrieve individual files from a server. These protocols offer a simple

							<h2>
								<span class="mw-headline" id="Static_website">Static website</span>
							</h2>
							<div role="note" class="hatnote navigation-not-searchable">Main article:
								<a href="/wiki/Static_web_page" title="Static web page">Static web page</a>
							</div>
							<p>
								A static website is one that has web pages stored on the server in the format that is sent to a client web browser. It is primarily coded in
									<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
								 (HTML);
									<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
									<a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a>
								 (CSS) are used to control appearance beyond basic HTML.  Images are commonly used to create the desired appearance and as part of the main content.  Audio or video might also be considered "static" content if it plays automatically or is generally non-interactive. This type of website usually displays the same information to all visitors. Similar to handing out a printed brochure to customers or clients, a static website will generally provide consistent, standard information for an extended period of time. Although the website owner may make updates periodically, it is a manual process to edit the text, photos, and other content and may require basic website design skills and software.  Simple forms or marketing examples of websites, such as
									<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
									<a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a>
									<i>classic website</i>
								, a
									<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
									<a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a>
									<i>classic website</i>
									<i>five-page website</i>
								 or a
									<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
									<a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a>
									<i>classic website</i>
									<i>five-page website</i>
									<i>brochure website</i>
								 are often static websites, because they present pre-defined, static information to the user. This may include information about a company and its products and services through text, photos, animations, audio/video, and navigation menus.

								<a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a>
								<a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a>
								<i>classic website</i>
								<i>five-page website</i>
								<i>brochure website</i>
